Данные программные средства позволяют строить различные перспективы потоков работ (в настоящий момент поддерживается перспектива потока управления и перспектива ресурсов). Modeler  отвечает за создание диаграмм, тогда как DiagramToSpin отвечает за перевод диаграмм созданных в modeler в PROMELA и исполнение в SPIN (внимание: для работы программы нужен установленный spin). Данное средство позволяет производить моделирование, верификацию и построение контр примера (в случае обнаружения ошибок). После создание диаграмм в Modeler необходимо запустить DiagramToSpin и выбрать папку с диаграммами, затем нажать кнопку «Convert» (перевод моделей в PROMELA). Покажется результат перевода. При нажатии на кнопку check произойдет проверка существующей модели на наличие ошибок синхронизации. В папке «Мой проект» есть пример (можно выбрать эту папку в DiagramToSpin и нажать «Convert»)
